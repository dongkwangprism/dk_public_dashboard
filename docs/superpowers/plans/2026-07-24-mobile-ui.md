# Mobile UI and PWA Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 같은 URL의 767px 이하 화면에서 현장용 모바일 대시보드와 투찰 분석 바텀시트를 제공하고 PWA 설치 정보를 추가한다.

**Architecture:** 기존 `App`이 데이터 조회, 캐시, 변환, 회사 선택, 영업 메모를 계속 소유한다. `useIsMobile`로 반환 UI만 분기하고, 모바일 표시·제스처는 `src/mobile/`에 격리하며 정렬과 D-day 표현은 테스트 가능한 순수 함수로 분리한다.

**Tech Stack:** React 19, Vite 7, Vitest, Testing Library, CSS, Web App Manifest

## Global Constraints

- 분기 기준은 정확히 `matchMedia("(max-width: 767px)")`이다.
- 768px 이상 데스크톱 UI와 우측 투찰 분석 드로어는 기존 동작을 유지한다.
- 모바일에는 `공고 / 영업 / 예정`만 렌더링하고 가로 스크롤 테이블을 사용하지 않는다.
- API 호출, bundle 변환, 캐시, 영업 메모, 투찰 분석 계산은 기존 로직을 공유한다.
- 주요 터치 타깃은 높이 44px 이상이며 하단 UI는 `env(safe-area-inset-bottom)`을 반영한다.
- PWA는 설치 메타데이터만 제공하며 서비스 워커와 오프라인 캐시는 추가하지 않는다.

```js
export const isMobileMediaQuery = "(max-width: 767px)";

export function mobileDdayTone(dday) {
  if (dday <= 3) return "red";
  if (dday <= 7) return "orange";
  return "green";
}

export function mobileBidOpacity(dday) {
  return dday > 10 ? 0.55 : 1;
}

export function mergeUpcomingRows(plans = [], specs = []) {
  const specRows = specs
    .map((row) => ({ ...row, kind: "spec" }))
    .sort((a, b) => a.dday - b.dday);
  const planRows = plans
    .map((row) => ({ ...row, kind: "plan" }))
    .sort((a, b) => a.dday - b.dday);
  return [...specRows, ...planRows];
}
```

- [ ] **Step 5: `main.jsx`에서 데이터 공유 분기 연결**

```jsx
function useIsMobile() {
  const [isMobile, setIsMobile] = useState(
    () => window.matchMedia(isMobileMediaQuery).matches
  );
  useEffect(() => {
    const media = window.matchMedia(isMobileMediaQuery);
    const onChange = (event) => setIsMobile(event.matches);
    media.addEventListener("change", onChange);
    return () => media.removeEventListener("change", onChange);
  }, []);
  return isMobile;
}
```

기존 반환 JSX는 `DesktopAppView`로 추출하지 않고 `isMobile ? <MobileApp ... /> : 기존 JSX` 형태로 감싸 데스크톱 변경량을 최소화한다. 모바일 탭 변경도 기존 `tab` 상태를 사용해 endpoint 지연 로딩 효과를 그대로 탄다.
