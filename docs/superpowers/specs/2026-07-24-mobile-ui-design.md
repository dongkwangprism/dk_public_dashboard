# 모바일 UI 및 PWA 설계

## 목표

현재 데스크톱 대시보드와 같은 URL에서 화면 폭이 767px 이하일 때 현장용 모바일 UI를 렌더링한다. 데스크톱 화면과 데이터 조회·가공 로직은 유지하며, 모바일에서는 공고 확인, 지역 예산 기반 영업, 발주계획·사전규격 확인에 집중한다.

## 범위

### 필수

- `matchMedia("(max-width: 767px)")` 기반 레이아웃 컴포넌트 분기
- 모바일 하단 탭 `공고 / 영업 / 예정`
- 회사 전환 칩, 새로고침, 현재 데이터 상태
- 테이블을 사용하지 않는 모바일 전용 카드 목록
- 공고 카드의 공고명, D-day, 발주기관, 예산, 투찰 분석 액션
- 공고 투찰 분석 바텀시트
- 지역 예산 영업 카드와 공유 영업 메모 진입
- 발주계획 및 사전규격 카드
- 44px 이상 주요 터치 타깃과 iOS 안전영역
- PWA manifest, 홈 화면 아이콘 192px/512px, Apple 메타 태그

### 제외

- 모바일 경쟁사 분석, 발주처 패턴, 공략 기회 화면
- 데스크톱 UI의 시각 또는 기능 변경
- 서비스 워커 기반 오프라인 캐시
- 앱스토어 배포

## 구조

기존 `App`이 회사, 키워드, API bundle, 로딩 상태, 선택 공고, 영업 메모를 계속 소유한다. `useIsMobile`은 화면 폭 변경을 구독하고, `App`의 반환부에서 `DesktopAppView` 또는 `MobileAppView`를 선택한다.

두 레이아웃은 다음 값을 공유한다.

- 회사와 회사별 키워드
- endpoint 지연 로딩 및 새로고침 함수
- `transformBundle` 결과
- 데이터 상태와 오류·캐시 상태
- 투찰 분석 계산 함수
- 영업 메모와 자사 투찰 이력 저장소

모바일 전용 컴포넌트는 표시와 모바일 상호작용만 담당한다. API 호출, 집계, 캐시, 저장 로직을 복제하지 않는다.

## 모바일 셸

- 상단 헤더: 현재 탭 제목, 긴급 공고 수, 새로고침
- 회사 칩: 제일테크/동광프리즘, 한 줄 배치
- 콘텐츠: 세로 카드 목록
- 하단 고정 탭바: 공고/영업/예정
- 콘텐츠 하단 여백: 탭바 높이와 `safe-area-inset-bottom`을 포함

모바일 탭과 endpoint는 다음과 같이 대응한다.

| 모바일 탭 | endpoint | 정렬 |
| --- | --- | --- |
| 공고 | `bids` | 개찰 D-day 오름차순 |
| 영업 | `lofin`, `plan` | 지연지수 내림차순 |
| 예정 | `plan`, `spec` | 사전규격 마감 임박 우선, 이후 발주 예정 |

## 카드

### 공고

- 공고명과 D-day 배지를 첫 줄에 배치
- 발주기관, 예산, 투찰 분석 버튼 순서
- 카드 전체를 눌러도 투찰 분석을 열고, 원문 링크는 별도 링크로 제공
- D-3 이하는 빨강, D-7 이하는 주황, 그 외 초록
- D-10 초과는 카드 불투명도 0.55

### 영업

- 영업 등급, 기관, 사업명, 집행잔액, 지연지수, 집행률
- 공유 영업 상태와 메모 편집 영역을 카드 내부에서 접고 펼침
- 물품성 사업을 기본 표시하며 기존 분류·메모 저장 로직을 공유

### 예정

- 사전규격은 의견 제출 마감 D-day, 기관, 공개일, 요약, 원문 액션 표시
- 발주계획은 예정일, 기관, 금액, 공고기관을 표시
- 두 유형을 구분하는 배지를 사용하고 임박 항목을 먼저 정렬

## 투찰 분석 바텀시트

- 높이 85dvh, 상단 모서리 16px, 배경 오버레이
- 상단 중앙 34x4px 드래그 핸들
- 닫기: 아래 방향 스와이프, 오버레이 탭, Escape, 브라우저 뒤로가기
- 열릴 때 history state를 추가하고 닫힐 때 정리
- 낙찰하한율은 `type="text"`, `inputMode="decimal"`, `pattern="[0-9.]*"`
- 기관 낙찰률 분포는 모바일용 가로 막대 최대 5개로 압축
- 권장 투찰 3구간과 예상 금액을 결론부로 강조
- 경쟁사 요약은 최대 2개 항목
- 이번 투찰 기록 버튼은 44px 이상
- 품목 전체 분포와 자사 과거 이력은 접이식으로 제공
- 시트 하단은 `safe-area-inset-bottom`을 반영

## PWA

- `manifest.webmanifest`에 이름, 축약명, 시작 URL, standalone 표시, 색상, 아이콘 등록
- `index.html`에 manifest, theme-color, Apple web app 메타 태그와 touch icon 등록
- 임시 아이콘은 조달 대시보드를 연상하는 문서·체크 모티프와 기존 파란색 계열을 사용
- 오프라인 동작은 이번 범위에 포함하지 않는다.

## 오류와 빈 상태

- 기존 endpoint별 로딩, 캐시, stale, 오류 판단을 모바일에서도 그대로 표시
- 카드가 없으면 탭별 빈 상태 문구 표시
- 일부 endpoint 실패 시 확보된 카드 데이터는 계속 표시
- 네트워크 갱신 중 기존 데이터가 있으면 목록을 유지

## 접근성과 상호작용

- 탭바와 시트에 적절한 `aria-label`, `aria-current`, dialog 속성 적용
- 모든 주요 액션의 최소 높이는 44px
- 포커스 가능한 버튼과 링크를 사용하고 카드 클릭과 내부 버튼 이벤트 충돌 방지
- 바텀시트가 열리면 배경 스크롤을 잠근다.

## 검증

- 프론트엔드 production build 성공
- 390x844 모바일 뷰포트에서 세 탭, 카드, 고정 탭바, 바텀시트 시각 확인
- 1280px 이상에서 기존 6개 탭과 우측 드로어가 유지되는지 확인
- 767px/768px 경계에서 레이아웃 분기 확인
- 바텀시트 오버레이, 스와이프, Escape, 뒤로가기 닫기 확인
- 터치 타깃, 가로 스크롤 부재, iOS 안전영역 확인
- manifest와 두 아이콘의 경로·크기 확인
